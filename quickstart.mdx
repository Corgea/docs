## Get Started


<Info>
  **Prerequisite** You should have an existing scanning tool like Semgrep or Snyk. If you don't have an existing tool, [get started](setup_scanner) in less than 5 mins here.  
</Info>
<Note>Corgea currently supports Python, Javascript and Typescript. See our [roadmap](roadmap).</Note>

<Steps>
  <Step title="Register with Corgea:">
    <Tip>
    Start in less than 10 seconds by [registering](https://corgea.com/registeration) today for free. No credit card required. 
    </Tip>
  </Step>
  <Step title="Send vulnerability report to Corgea">
    After registering, you'll be welcomed with our Quickstart modal. 

    <Frame>
      <img src="/images/get-command.png" style={{ borderRadius: '0.5rem' }} />
    </Frame>


    Pick Snyk or Semgrep to get the command you need to run in the repository you want to fix. 

    The command you copy will have all the needed functions and your API key.


    <CodeGroup>
    ```bash Semgrep
    export CORGEA_TOKEN=<API_KEY>; curl -sSL https://cli.corgea.app | bash -s semgrep ci --json
    ```

    ```bash Snyk
    export CORGEA_TOKEN=<API_KEY>; curl -sSL https://cli.corgea.app | bash -s snyk code test --json
    ```
    </CodeGroup>


    You can alternatively click on "Load Demo" or use following projects are designed to test security tools. Try cloning and fixing them with Corgea. .


    <AccordionGroup>
      <Accordion title="Example vulnerable projects">
      juice-shop, a vulnerable Node.js + Express app:
      ```bash Bash
      git clone https://github.com/bkimminich/juice-shop
      cd juice-shop
      ```

      Vulnerable-Flask-App, vulnerable Python + Flask:
      ```bash Bash
      git clone https://github.com/we45/Vulnerable-Flask-App
      cd Vulnerable-Flask-App
      ```

      </Accordion>
    </AccordionGroup>

  </Step>
  <Step title="Issue Fixes">
    These are instructions or content that only pertain to the third step.
  </Step>
</Steps>
